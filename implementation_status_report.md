# ITS VVC 逆变换系统 - 实现状态与赛题要求对比报告

## 1. 项目概述

本报告对比 VVC (H.266) 逆变换系统 (ITS) 的当前实现状态与第九届中国研究生创芯大赛华为赛题1的技术要求。

---

## 2. 接口对比

### 2.1 赛题要求（Version B - 最终版）

| 信号 | 位宽 | 方向 | 说明 |
|------|------|------|------|
| `it_info` | 22-bit | I | [6:0] width, [13:7] height, [15:14] tr_type_hor, [17:16] tr_type_ver, [19:18] lfnst_tr_set_idx, [21:20] lfnst_idx |
| `it_info_vld` | 1-bit | I | 脉冲有效 |
| `it_data_in` | 16-bit | I | 有符号输入数据 |
| `it_data_addr` | 12-bit | I | TU 内光栅扫描位置 |
| `it_data_end` | 1-bit | I | 数据输入结束脉冲 |
| `it_data_in_vld` | 1-bit | I | 输入数据有效 |
| `it_data_out` | 40-bit | O | 4×10-bit 输出 |
| `it_data_out_vld` | 1-bit | O | 输出有效 |
| `it_done` | 1-bit | O | 计算完成脉冲 |

### 2.2 当前实现

| 信号 | 位宽 | 状态 | 说明 |
|------|------|------|------|
| `it_info` | 22-bit | **符合** | 位域分配正确 |
| `it_info_vld` | 1-bit | **符合** | - |
| `it_data_in` | 16-bit | **符合** | - |
| `it_data_addr` | 12-bit | **符合** | - |
| `it_data_end` | 1-bit | **不符合** | 使用超时检测代替（idle_cnt >= 15） |
| `it_data_in_vld` | 1-bit | **符合** | - |
| `it_data_in_req` | 1-bit | **多出** | 赛题无此信号（Version B） |
| `it_data_out` | 40-bit | **符合** | 4×10-bit 有符号 |
| `it_data_out_vld` | 1-bit | **符合** | - |
| `it_data_out_req` | 1-bit | **多出** | 赛题无此信号（Version B） |
| `it_done` | 1-bit | **符合** | - |

**接口差异总结：**
- 缺少 `it_data_end` 输入信号
- 多出 `it_data_in_req` 和 `it_data_out_req` 反压信号（属于 Version A 规范）
- 当前实现使用超时机制检测输入结束，而非显式 `it_data_end` 信号

## 3. 功能对比

### 3.1 变换类型支持

| 变换类型 | 赛题要求 | 当前状态 | 说明 |
|----------|----------|----------|------|
| DCT2 | 必须 | **支持** | 25 种尺寸全部支持 |
| DCT8 | 必须 | **支持** | 16 种尺寸全部支持 |
| DST7 | 必须 | **支持** | 16 种尺寸全部支持 |
| LFNST | 必须 | **部分支持** | 见下文详细分析 |

### 3.2 支持的块尺寸

| 类型 | 赛题要求 | 当前支持 | 状态 |
|------|----------|----------|------|
| DCT2 | 4×4 ~ 64×64 (25种) | 4×4 ~ 64×64 | **全部支持** |
| DCT8 | 4×4 ~ 32×32 (16种) | 4×4 ~ 32×32 | **全部支持** |
| DST7 | 4×4 ~ 32×32 (16种) | 4×4 ~ 32×32 | **全部支持** |

### 3.3 LFNST 支持分析

| 要求项 | 赛题要求 | 当前状态 | 差距 |
|--------|----------|----------|------|
| lfnst_idx | 0/1/2 (0=不启用) | **支持** | - |
| lfnst_tr_set_idx | 0/1/2/3 | **未使用** | 接口已有但未接入逻辑 |
| nTrs 计算 | `(w>=8 && h>=8) ? 48 : 16` | 使用 nonZeroSize=8/16 | **定义错误** |
| nTrs=16 矩阵 | 16×16 (8场景) | 8×8 矩阵 | **维度错误** |
| nTrs=48 矩阵 | 48×16 (8场景) | 16×8 矩阵 | **维度错误** |
| ROM 条目数 | 8192 条 | 384 条 | **严重不足** |
| 场景数 | 16个 (4 set_idx × 2 idx × 2 nTrs) | 4个 | **缺少12个场景** |

**LFNST 是当前最大的功能缺陷**，需要完全重写 LFNST ROM 和模块。

### 3.4 数据流

| 要求项 | 赛题要求 | 当前状态 | 状态 |
|--------|----------|----------|------|
| 输入顺序 | 光栅扫描，逐点 | **符合** | - |
| 输出顺序 | 光栅扫描，4点/周期 | **符合** | - |
| 输出位宽 | 10-bit 有符号 | **符合** | Clip3(-512, 511) |
| 处理顺序 | 先水平后垂直 | **符合** | 行变换 → 列变换 |
| 非零输入 | 只传非零点 | **符合** | - |

## 4. 性能对比

### 4.1 时序性能

| 指标 | 赛题要求 | 当前实现 | 状态 |
|------|----------|----------|------|
| 工作频率 | **500 MHz** | **133 MHz** (Artix-7) | **不满足** |
| 计算吞吐 | 4点/周期 | 4点/周期 | **满足** |
| 流水线支持 | 必须 | 2级 DSP 流水线 + 2级输出流水线 | **满足** |

### 4.2 时序瓶颈分析

| 关键路径 | 延迟 | 逻辑级数 | 说明 |
|----------|------|----------|------|
| LFNST 写地址 → in_mem | 6.975ns | 9级 | 最大瓶颈 |
| BRAM → OBUF (已解决) | 5.124ns | 1级 | 输出流水线已修复 |

**要达到 500MHz (2ns 周期)：**
- 当前关键路径 6.975ns 需要减少到 < 2ns
- 需要对 LFNST 写地址路径进行 3-4 级流水线改造
- 或切换到更快的 FPGA 器件 (UltraScale+ 可能达到 300-400MHz)

### 4.3 资源利用 (Artix-7 xc7a200tfbg484-2)

| 资源 | 使用量 | 可用量 | 利用率 |
|------|--------|--------|--------|

| LUT as Logic | 2,663 | 134,600 | 1.98% |
| LUT as Memory | 4,352 | 46,200 | 9.42% |
| Slice Registers | 2,384 | 269,200 | 0.89% |
| DSP48E | 8 | 740 | 1.08% |

### 4.4 功耗 (Artix-7, 500MHz 约束)

| 类型 | 功耗 |
|------|------|
| 动态功耗 | 0.579 W |
| 静态功耗 | 0.133 W |

## 8. 关键差距总结

### 8.1 必须修复（影响功能正确性）

1. **LFNST 模块重写** - 当前实现严重不符合赛题规范
   - `lfnst_tr_set_idx` 未使用
   - `nTrs` 定义错误
   - 矩阵维度错误
   - ROM 严重不足（需要 8192 条，当前 384 条）

2. **接口修复** - 添加 `it_data_end` 信号
   - 删除 `it_data_in_req` 和 `it_data_out_req`（Version B 规范）
   - 或保留反压信号（Version A 规范，需确认）

### 8.2 建议优化（影响评分）

3. **时序优化** - 目标 500MHz
   - LFNST 写地址路径流水线化
   - 考虑更快的 FPGA 器件
   - 或接受较低频率，优化 PPA

4. **文档编写** - 设计文档 + 验证报告 + PPA 报告

5. **波形截图** - 关键场景的仿真波形
